    Introduction

    Welcome to the Credino API Documentation! This documentation will guide you on how to use the Credino API.

    Authentication

    To authorize, use this code:

    # This example is to show how to use Bearer Authorization header
curl "credino_api_endpoint"
  -H "Authorization: Bearer 123123123123"

    Credino API uses Authorization Bearer header as an authorization to access the endpoints.

    1. Please send Bearer Authentication header which can be obtained when login to the app. To get this key, please contact the administrator. Afterward, use the token data returned from dashboard login. It should be included in all API requests in a header that looks like the following:

      Authorization: Bearer 123123123123

    You must replace 123123123123 with your personal API key.

    This endpoint retrieves Credino's User credit score that has been normalized using linear regression AI engine.

    This endpoint can only be accessed using Bearer Authorization Header.

    HTTP Request

    GET https://api.credino.io/credit-scores/{userID}

    Path Parameters

    Parameter Data Type Description
    userID (Required) uint64 Credino's user ID

    This endpoint retrieves list of dashboard roles.

    This endpoint can only be accessed using Bearer Authorization Header.

    HTTP Request

    POST "https://api.credino.io/credit-scores/disputes/{userID}"

    Query Parameters

    Parameter Description
    role_name role name to be filtered
    page requested page.
    limit requested limit to set the number of data to return.
    order_by to sort the result, e.g. id ASC

    Users

    Register New User

    curl -X POST "https://api.credino.io/users/register"
  -H "Authorization: Bearer 123123123123"

    Sample Request Payload:

    {
    "name": "Foo Bar",
    "email": "test@mail.com",
    "phone": "+62812123123113"
}

    Sample JSON successful response:

    {
  "success": true,
  "message": "success",
  "data": {
    "id": 1,
    "name": "Foo Bar",
    "email": "test@mail.com",
    "phone": "+62812123123113"
  }
}

    Sample JSON response with errors:

    {
  "success": false,
  "message": "email already exist",
  "error": "failed to register user"
}

    This endpoint will register a new user to Credino system.

    This endpoint can only be accessed using Bearer Authorization Header.

    HTTP Request

    POST "https://api.credino.io/users/register"

    Request Body Parameters

    Parameter Data Type Description
    name (Required) string Credino user's name
    email (Required) string Credino user's email
    phone (Required) string Credino user's phone number

    Request OTP

    curl -X POST "https://api.credino.io/users/otp"
  -H "Authorization: Bearer 123123123123"

    Sample Request Payload:

    {
    "user_id": 1,
    "otp_type": "email"
}

    Sample JSON successful response:

    {
  "success": true,
  "message": "success",
  "data": null
}

    Sample JSON response with errors:

    {
  "success": false,
  "message": "already requested otp 5 times. please try again later",
  "error": "failed to request otp"
}

    This endpoint will send OTP to user through email or SMS.

    This endpoint can only be accessed using Bearer Authorization Header.

    HTTP Request

    POST "https://api.credino.io/users/otp"

    Request Body Parameters

    Parameter Data Type Description
    user_id (Required) uint user_id
    otp_type (Required) string otp type (one of: phone, email)

    This endpoint can only be accessed using Bearer Authorization Header.

    HTTP Request

    POST "https://api.credino.io/users/upload-id"

    Errors

    Credino API uses the following error codes:

    Error Code Meaning
    200 Successful response when getting, updating, or deleting data
    201 Created -- data successfully created.
    400 Bad Request -- Your request is invalid or missing parameters.
    401 Unauthorized -- Your API key is wrong or missing API key.
    403 Forbidden -- The endpoint can only be accessed using certain API key.
    500 Internal Server Error -- Something wrong with the query or we had a problem with our server. Try again later.
